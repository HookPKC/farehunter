# FareHunter Prompt 模板集

> 開新輪次時複製對應模板，填入〈〉內容。搭配 `PLAYBOOK_SHORT.md` 使用。
> 每個模板都內建本專案的防呆限制，換其他 AI 工具也適用。

---

## 1. 調查型（只查不改）

```
不要修改任何程式、不要部署。只做調查。
主題：〈要查什麼，例：站上 X 欄位的真正語意〉
限制：
1. 不要重分析整個專案
2. 只看與〈主題〉直接相關的檔案，先列出你要看哪些
3. 結論要附證據（檔案行號、線上實際資料數字）
請回答：
1. 〈問題一〉
2. 〈問題二〉
3. 最後給結論：目前可接受 / 需文案修正 / 需 deeper 修正
只輸出調查報告。
```

## 2. 提案型（查完先提案，等確認）

```
先不要實作，只提出修正方案。
背景：〈上一輪調查結論或問題描述〉
目標：〈要達成什麼，不是要做什麼功能〉
限制：
1. 不要動 Recommendation Engine / 資料模型 / 票價來源 / Google 深連結（依需要增減）
2. 不要大改版
3. 先列出準備修改的檔案與區塊
4. 每項列修改前 / 修改後（文案或行為）
5. 若會影響其他功能或需要動到範圍外檔案，明確標示風險，不要直接納入
6. 分成 P0 必修 / P1 建議 / P2 可改善（若適用）
等我確認後你再實作。
```

## 3. 小改動實作型（低風險直接做）

```
照〈已確認的提案／以下描述〉實作，最小修改。
只允許修改：〈檔案清單，例：docs/index.html〉
限制：
1. 不要重分析專案；續作先跑 git status + git diff
2. 不要動〈禁區，例：gfLink / 引擎 / 通知邏輯〉
3. 改文案要同步 grep 並更新鎖舊字串的測試
4. 若發現需要改到允許清單以外，先停下來回報，不要自行擴大
完成後只回報：
1. 修改了哪些檔案
2. 修改前 / 修改後差異
3. 驗證方式與結果
4. 是否還有剩餘已知限制（含你無法驗證的部分）
```

## 4. 驗收型（改完的獨立檢查）

```
不要修改、不要部署，只做驗收。
對象：〈上一輪完成的改動〉
請驗證：
1. pytest 全綠、smoke 通過
2. 用線上實際 data.json / ranked.json 做全頁渲染，列出具體輸出
   （實際順序 / 實際字串 / 實際數字），逐項與預期比對
3. google 來源與快取來源各挑一個案例分別檢查
4. 若探針回報 ✗，先驗證探針本身（印原始字串 / charCode / grep 元素 id）
   再下結論
只回報：通過清單、未通過項目與原因、需要我人工驗證的項目。
```

## 5. 回退型（實測退化時）

```
只做回退，不要重分析專案，不要改其他功能。
問題：〈實測到的退化現象，附證據〉
要求：
1. 只修改〈檔案〉的〈函式/區塊〉，恢復為先前可用版本
2. 不要順手修其他東西
3. 在程式內留簡短註解記錄：為什麼回退、之後不要再用哪種做法
4. 若專案有已知限制文件（README / PLAYBOOK），補一小段紀錄
完成後列出：回退後的實際行為範例〈例：3 組最終 URL〉、
是否已恢復正常、剩餘限制。
```

## 6. 部署 / live 驗證型

```
只做部署與線上驗證，不要改任何程式。
步驟：
1. 確認工作目錄乾淨（git status），commit 訊息如實描述
2. push 後等 30 秒以上再 dispatch deploy（或等 workflow_run 自動觸發）
3. 驗收必核對：
   a. deploy run 的 head_sha == main HEAD
   b. deployments?environment=github-pages 最新一筆的 sha == main HEAD
   c. 不得以「run success」或「repo raw 檔有新 code」代替 a/b
4. 若涉及外部服務行為（Google 落地頁 / LINE 通知 / 手機排版），
   標記為「待我真機驗收」，不要宣告完成
回報：部署的 SHA、驗收結果、需要我人工做的最後確認（例：無痕視窗開一次）。
```

## 7. 給新工具/新模型的開場模板

```
你將接手 FareHunter 專案的一個任務。開始前先讀 docs/PLAYBOOK_SHORT.md，
嚴格遵守其中 10 條守則與 5 個高風險區。
特別注意：
- gfLink 的 q= 已凍結（實測過加參數會退化），不要動
- 價格顯示原則：google 觀測價精確、快取一律「約 NT$X」百位化
- 快取來源信心永不 High、Best 可能比 cheapest 貴——這些是刻意設計，不要「優化」
- 任何語意類改動先提案等我確認
本輪任務：〈任務〉
```
